# ai_glossary

20 terms worth knowing

Machine Learning

Teaching a computer to find patterns in data so it can make decisions without being told every rule. Instead of programming the answer, you show it examples.

Algorithm

A step-by-step set of instructions a computer follows to solve a problem. Every model runs on an algorithm under the hood.

Model

The thing an ML system produces after learning from data. You feed it new inputs and it returns a prediction. The model is what gets deployed.

Training

The process of showing a model lots of labelled examples so it can learn the patterns between inputs and outputs. Training only happens once; prediction happens forever after.

Feature

A piece of input information the model uses to make its prediction. For a song: tempo, energy, danceability. For a video: platform, category, creator tier.

Label

The answer the model is trying to predict. "Hit or flop." "Pass or fail." "Rising, stable, declining, or seasonal." Labels are what you give the model during training.

Supervised Learning

Training where you give the model both the inputs and the correct answers. The most common type of ML. Used whenever you have labelled historical data.

Unsupervised Learning

Training with no labels. The model finds structure on its own by grouping similar things together. Used when you do not know what categories exist yet.

Classification

Sorting inputs into categories. Spam or not spam. Trending or declining. Pass or fail. When the output is a named category, it is a classification problem.

Regression

Predicting a number rather than a category. "What score will this student get?" or "How many views will this video get?" When the output is continuous, it is regression.

Accuracy

The percentage of predictions the model got right. Sounds simple, but misleading when one class is much more common than the others. Always check the confusion matrix too.

Confusion Matrix

A grid that shows exactly where a model's predictions were right and where they went wrong. Rows are the real answers. Columns are what the model predicted. The diagonal is correct.

Overfitting

When a model memorises the training data instead of learning general patterns. It scores brilliantly on data it has seen and fails on anything new. The ML equivalent of cheating on a practice paper.

Bias in AI

When a model systematically favours or penalises certain groups, usually because the training data did not represent everyone equally. A model trained mostly on one demographic makes worse predictions for everyone else.

Neural Network

A type of model loosely inspired by how the brain connects neurons. Powerful for images, speech, and language. The foundation of most modern AI you interact with daily.

Large Language Model (LLM)

The type of model behind ChatGPT, Claude, and Gemini. Trained on enormous amounts of text to predict what comes next. That prediction loop, done billions of times, produces what feels like conversation.

Data Leakage

Using information during training that would not actually be available at prediction time. The model looks brilliant in testing but useless in the real world. One of the most common and costly ML mistakes.

Clustering

Grouping similar examples together without being told what the groups should be. K-means is the classic algorithm. Used to find natural structure in data before you know what to look for.

Precision and Recall

Two measures that matter when one type of mistake is worse than another. Precision: of the things the model flagged, how many were real? Recall: of all the real ones, how many did the model catch?

CRISP-DM

A six-phase framework for running ML projects: understand the business problem, understand the data, prepare the data, build the model, evaluate it, then deploy. Used by professionals worldwide.
